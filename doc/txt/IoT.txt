--avr--
The AVR is a "modified Harvard architecture" machine 
where program and data are stored in separate physical 
memory systems that appear in different address spaces, 
but having the ability to read data items from 
program memory using special instructions.
哈佛和冯诺依曼主要区别是否区分指令与数据
内存里指令和数据在一起
CPU区分指令缓存和数据缓存，指令和数据从两个不同的地方出来
可以理解为CPU外部，是冯诺依曼，内部是哈佛
哈佛结构复杂，效率高
冯诺依曼简单，效率底
为了提高速度，CPU增加指令缓存和数据缓存，为解决现实问题，什么主义不重要
指令和数据共享同一 address space，但缓存分开，是两种架构的折中
